Check rollout status:

  kubectl rollout status -n {{ .Release.Namespace }} deployment/{{ include "soc-platform.fullname" . }}-api

  kubectl rollout status -n {{ .Release.Namespace }} deployment/{{ include "soc-platform.fullname" . }}-web

Database migrations are NOT run by this chart — apply them yourself before
the first install against a new database (or as a release hook if you adopt
one later):

  DATABASE_URL=<your-url> pnpm --filter @soc/database exec prisma migrate deploy

Roll back a bad release:

  helm rollback {{ .Release.Name }} -n {{ .Release.Namespace }}
